# Problem Solving Log

## Overview

A record of reusable solution patterns, failed approaches, and the reasoning that turns individual exercises into transferable knowledge.

## Key Concepts

- Recognizing structural patterns before choosing an algorithm.
- Writing down invariants and boundary conditions.
- Comparing brute-force, optimized, and alternative solutions.
- Reviewing mistakes after implementation.

## Notes

Each future entry will record the problem shape, first approach, failure mode, final insight, complexity, and a concise implementation note.

## References

- Problem sets and source links will be attached to individual entries.
